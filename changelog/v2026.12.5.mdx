## Document Analyzer and Reliability Improvements

This is a broad release touching several systems. The headline features are a new document analyzer agent, multi-currency payment support, and fixes to scheduler deadlocks and storage memory pressure. The auth middleware was rewritten as pure ASGI, the UI got accessibility improvements, and 44 new tests were added.

---

## What You Need to Change

<Warning>
**Stricter configuration validation.** Deployments missing `deployment.url` or nested required fields will now fail fast on startup instead of failing silently later. Make sure your `agent_config.json` includes all required deployment fields.
</Warning>

<Warning>
**Constants reorganized.** If you import constants from deeply nested paths in the extensions modules, check that your imports still resolve. Key settings moved to dedicated constants modules.
</Warning>

---

## What's New

### Document Analyzer Agent

A new document analyzer agent with skills wiring and workflow examples for processing PDF and DOCX files. The frontend restricts uploads to allowed mimetypes and ensures `FilePart.text` is preserved through the pipeline so downstream handlers can operate on the extracted content.

### Multi-Currency Payments

The payment system now supports multiple currencies and multi-asset requirements in a single configuration. Payment context validation was tightened to prevent duplicate injection and partial state corruption. The A2A `message/send` endpoint no longer returns 500 errors when payment states are malformed.

### DSPy and LangGraph Examples

A new DSPy agent example demonstrates question-answering behavior with comprehensive unit tests. A LangGraph structured workflow example shows advanced orchestration patterns within Bindu.

### UI and Accessibility

A collapsible Agent Inspector with clearer empty states improves the first-agent experience. Skip-to-content links and ARIA landmarks improve screen reader navigation. Visual flicker when switching between previous chats has been eliminated.

---

## What's Fixed

**Scheduler** — Resolved AnyIO buffer deadlocks, CPU burn loops in edge cases, and trace serialization failures with large payloads.

**Storage** — Hardened the in-memory subsystem to reduce OOM risks and optimized database indexing.

**Auth middleware** — Migrated the Hydra middleware from synchronous to pure ASGI. This improves concurrency, adds WebSocket support, and properly manages connection lifecycle to prevent resource leaks.

**Configuration** — Added fail-fast validation for `deployment.url` with much clearer error messages for missing nested required fields.

---

## What's Improved

### Test Coverage

44 new tests were added across critical components:

- **ManifestWorker** (+15 tests) — Task execution flows, payment context integration, error handling, system message injection. Coverage: 58% → 83%.
- **BinduApplication** (+20 tests) — Configuration validation, custom setup, debug mode, CORS, authentication. Coverage: 16% → 30%.
- **TaskManager** (+9 tests) — Context manager, storage/scheduler attributes, push manager initialization. Coverage: 67% → 75%.

Overall coverage improved from 58.59% to 60.11% with 688 tests passing.

## Migration

<Steps>
  <Step title="Check your config">
    Ensure `deployment.url` is properly set. The validator is now strict about this.
  </Step>
  <Step title="Update payment configs">
    If you use custom payment configurations, verify they map correctly to the new multi-currency validation rules.
  </Step>
  <Step title="Rebase custom extensions">
    If you maintain forks of the scheduler, storage, or Hydra auth modules, rebase onto this release to pick up the deadlock and ASGI safety fixes.
  </Step>
</Steps>
