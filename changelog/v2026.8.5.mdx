## Code Minimization and Observability Refactor

This release removes ~590 lines of unused code from the observability, x402, and DID extension modules. The goal was to shrink the surface area — fewer code paths means fewer things to maintain, understand, and secure.

Production functionality is 100% preserved. However, if you were importing Sentry wrapper functions from `bindu.observability` or x402 constants from `bindu/extensions/x402/constants.py`, those have moved or been removed.

---

## What You Need to Change

<Warning>
**Sentry wrapper functions removed.** `capture_exception()`, `capture_message()`, `set_user()`, `set_context()`, `add_breadcrumb()`, and `start_transaction()` are no longer exported from `bindu.observability`. Use `sentry_sdk` directly instead:

```python
# Before
from bindu.observability import capture_exception, set_user

# After
import sentry_sdk
sentry_sdk.capture_exception(error)
sentry_sdk.set_user({"id": user_id})
```
</Warning>

<Warning>
**x402 constants file deleted.** Constants moved to `bindu.settings.X402Settings`:

```python
# Before
from bindu.extensions.x402.constants import X402_EXTENSION_URI

# After
from bindu.settings import app_settings
app_settings.x402.extension_uri
```
</Warning>

---

## What's Improved

The observability module went from 8 exports to 2 (`setup` and `init_sentry`). The x402 extension dropped unused utility functions (`build_payment_required_metadata`, `build_payment_verified_metadata`, `merge_task_metadata`, `get_agent_extension`). DID extension helper methods were inlined for simpler code flow.

The Alembic migration runner now auto-converts `postgresql://` URLs to `postgresql+asyncpg://`, so both sync and async PostgreSQL URLs work without manual configuration:

```bash
# Both formats now work
DATABASE_URL=postgresql://user:password@host/db alembic upgrade head
DATABASE_URL=postgresql+asyncpg://user:password@host/db alembic upgrade head
```

---

## Migration

If you use `sentry_sdk` features through Bindu's wrappers, switch to importing `sentry_sdk` directly. If you reference x402 constants, switch to `app_settings.x402.*`. Everything else — OpenInference setup, Sentry initialization, DID extension behavior, and database migrations — works the same as before.
