## Bug Fixes and Community Contributions

Stability release with fixes for schema management, type checking, and dependency compatibility. Also adds three new example agents contributed by the community. No breaking changes.

---

## What's Fixed

**Schema management** — Schema name truncation now correctly caps at 63 characters (PostgreSQL's limit). Previously, long DID-based schema names could exceed this.

**Type checking** — Fixed a type checker error for `sys.stdout.reconfigure` and resolved type checking issues across several modules.

**Task manager** — Stabilized `task_manager.py` with improved error handling and edge case coverage.

**Dependencies** — Updated pydantic (2.11.3 → 2.12.5), requests (2.32.3 → 2.32.5), sqlalchemy (2.0.44 → 2.0.46), tenacity (8.5.0 → 9.1.4), and rich (13.9.4 → 14.3.2). The pydantic update resolves a fastmcp compatibility issue; the requests update resolves a langchain-community compatibility issue.

---

## What's New

### Example Agents

Three new examples contributed by the community:

- **Paywall Agent** — Demonstrates x402 payment integration, showing how to monetize agent responses
- **News Summarizer** — Uses local Ollama and DuckDuckGo for privacy-friendly news aggregation
- **Cybersecurity Newsletter** — Security-focused agent for hardening best practices

### Validation

The `author` field is now validated early in the configuration pipeline. Missing it produces a clear error at startup instead of a confusing failure later.

### Observability

`NotificationService` now tracks lightweight in-memory delivery metrics, and the health endpoint has improved observability.

---

## Migration

If upgrading from a version with exact dependency pins, run `uv sync` to pick up the updated packages. Make sure your agent config includes the required `author` field.
