## UI Separation and Test Fixes

Maintenance release. The built-in documentation UI has been moved to a separate frontend application, and the `/docs` and `/favicon.ico` endpoints have been removed from the core server. This cleans up the boundary between the backend API and the presentation layer.

---

## What You Need to Change

<Warning>
**Built-in UI endpoints removed.** The `/docs` endpoint no longer exists. If you were using it, switch to the separate frontend application (typically at `http://localhost:5173`).
</Warning>

All API endpoints remain unchanged — `POST /` for A2A protocol, `GET /health`, `GET /metrics`, `GET /.well-known/agent.json`, and `GET /agent/skills` all work exactly as before.

---

## What's Improved

Test infrastructure was cleaned up alongside the UI removal. Thirty previously failing tests in `test_applications.py` were fixed, deprecated endpoint tests were removed, and the route structure assertions were updated to reflect the new, leaner set of core routes.

Example files were standardized with consistent `launch=True` patterns and tunnel launch comments. A new `echo_agent_behind_paywall.py` example demonstrates x402 payment integration. Documentation credentials were sanitized with `pragma: allowlist secret` comments to prevent false positives from the detect-secrets pre-commit hook.

---

## Migration

If you were accessing the built-in UI at `http://localhost:3773/docs`, that path is gone. Use the separate frontend application instead. All agent API endpoints remain at their existing paths.
