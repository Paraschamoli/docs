## Prometheus Metrics and Monitoring

This release adds a `/metrics` endpoint that speaks Prometheus text format. If you run your agents in production, you can now scrape them with Prometheus and build Grafana dashboards without any extra dependencies — the implementation is pure Python.

No breaking changes. The endpoint is available immediately after upgrading.

---

## What's New

### The `/metrics` Endpoint

Every Bindu agent now exposes a Prometheus-compatible metrics endpoint. A middleware layer automatically tracks HTTP requests, and the endpoint aggregates everything in a thread-safe collector.

Here is what gets tracked:

| Type | Metrics |
|------|---------|
| Counters | `http_requests_total` (by method, endpoint, status), `agent_tasks_completed_total`, `agent_errors_total` |
| Histograms | `http_request_duration_seconds` (0.1s, 0.5s, 1.0s, +Inf), `task_duration_seconds` (1s, 5s, 10s, 30s, 60s, +Inf) |
| Gauges | `agent_tasks_active` (per agent), `http_requests_in_flight` |
| Summaries | `http_request_size_bytes`, `http_response_size_bytes` |

### Quick Start

Scrape your agent:

```bash
curl http://localhost:3773/metrics
```

Point Prometheus at it:

```yaml
scrape_configs:
  - job_name: 'bindu-agent'
    static_configs:
      - targets: ['localhost:3773']
    metrics_path: '/metrics'
```

### Useful PromQL Queries

```promql
# Request rate
rate(http_requests_total[5m])

# 95th percentile latency
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))

# Error rate
rate(agent_errors_total[5m])

# Average task duration
rate(task_duration_seconds_sum[5m]) / rate(task_duration_seconds_count[5m])
```

## What's Improved

The middleware is positioned last in the stack so it captures the complete request lifecycle, including response sizes and in-flight counts. The metrics module has no circular dependencies and a clean three-file structure: core collection, HTTP middleware, and the Prometheus endpoint.

---

## Migration

No action required. Metrics are automatically enabled. Access them at `/metrics` and optionally configure Prometheus to scrape your agent.
